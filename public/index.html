What am I looking at? — sections & how they're calculated
🗼 Live Dashboard
What: real-time snapshot of every agent — online/offline counts, tokens, cost, average latency and active alerts.
How: built from agent heartbeats. Hover a card for the per-agent breakdown; click it to expand with filters. An agent counts as offline if no heartbeat for ~45s.
🕸 Lineage graph
What: how agents relate — each agent, the tools it exposes, and agent → agent interdependencies.
How: interdependency arrows are derived from who triggered whom (trigger runs) plus any declared depends_on. Tools come from each agent's reported tool list / an MCP's tools/list.
⚠ Alerts
What: automatic anomaly detections per agent.
How: Token spike — window tokens > 2.5× the agent's rolling baseline (EMA). Tool loop — same tool called ≥5× in one report window. High error rate — error_rate > 15%. Silent failure — no heartbeat for >60s.
🙋 Escalations
What: human-in-the-loop approvals. An agent pauses and asks a person to approve / reject / hold a decision.
How: the agent raises an escalation and polls for the outcome; you resolve it here and the agent continues with your decision.
📜 Audit log
What: a time-ordered trace of everything an agent reported.
How: each event (task start/complete, tool calls, LLM responses, triggers, escalations) is appended with its token count as it arrives.
💲 Cost attribution
What: spend per agent and per department.
How: for every LLM response, cost = (tokens_in ÷ 1000 × input_rate) + (tokens_out ÷ 1000 × output_rate). Rates are per-model from cost-config.json; tokens and cost accumulate across heartbeats.
